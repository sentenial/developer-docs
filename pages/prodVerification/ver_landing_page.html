Introduction

Verifying account ownership before disbursing funds helps to prevent fraud and minimise the risk of misdirected payments. Verification allows you to validate beneficiary details prior to payment execution.

Nuapay allows you to verify account holders in two ways, via:

  • Confirmation of Payee (CoP) / Verification of Payee (VoP)
  • Open Banking - AIS
{% include warning.html content="Verification via Open Banking AIS will be deprecated in November 2025 so we recommend using CoP (for UK clients) or VoP as your verification method." %}

CoP and VoP

Account verification services, including Confirmation of Payee (CoP) in the UK and Verification of Payee (VoP) in the EU and other regions, allow payers or systems to check that a payee's name matches the account details provided. These services help prevent typographical errors, misdirected payments, and fraud before funds are transferred.

Term Region Purpose Description
Confirmation of Payee (CoP) UK Verify payee details Confirms that the name on a bank account matches the account number before making a payment, helping to reduce fraud and misdirected payments.
Verification of Payee (VoP) EU / SEPA Verify payee details Checks that the recipient’s name matches the IBAN and account details before a payment is processed, reducing errors and unauthorized transactions.

Common Verification Results

  • Match: The payee name exactly matches the account details. Payment can proceed with confidence.
  • Partial Match: The payee name partially matches the account details (e.g., minor spelling differences). Users or systems may require additional confirmation.
  • No Match: The payee name does not match the account details. Payment should be stopped or reviewed to avoid errors or potential fraud.

These results are common across both CoP and VoP implementations, though specific handling, terminology, and regulatory requirements may vary by region.

Why these checks matter

  • Prevent misdirected payments and reduce refund workload.
  • Mitigate authorised push payment (APP) and other account-targeted fraud.
  • Improve customer confidence by displaying verification results before payment confirmation.

AIS Mode

Account Holder Verification allows you to leverage the power of Open Banking Account Information Service to verify if user details, held by you, match the account details stored at the user's bank.

{% include tip.html content="Want to start testing? See the Getting Started section to request an API Key for the Nuapay Sandbox." %}

Open Banking Account Information Service (AIS) allows merchants, via the Nuapay TPP, to verify account ownership of individuals. The solution has been designed to be flexible enough to be used for various use cases.

For example:

  • A merchant may want to issue a payout (via the Create Credit Transfer & Beneficiary API) but first wants to confirm that the customer name that the merchant holds is the same as the account name into which the payment will be made.
  • If the user is Tommy Smith, for example, the merchant can use the Verification service to request the user to grant consent to view account details at his bank.
  • If the account holder name returned by the bank is Thomas Smith, the name matching algorithm will result in a 100% match, so the merchant can be confident that the payment that is about to be made is going to go to the correct beneficiary
{% include note.html content="Name matching will return a match score; 100% is used for exact or very close matches. Merchants will need to determine what scores are acceptable and will need to determine the risk and value of transaction when deciding on how to proceed where the verification matching score is less than 100%" %}

Customer verification is a 2-step process:

  • Request the Customer consent to retrieve account details from the bank.
  • Using the retrieved account details, calculate a verification score against data that the client already holds (i.e., the customer’s name). Currently the matching is carried out only on the customer’s name.
